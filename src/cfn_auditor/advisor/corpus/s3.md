# S3 Bucket Hardening (CIS 2.x; Well-Architected SEC-08)

S3 buckets must encrypt objects at rest and reject public access by default.

- **Encryption at rest** — set `BucketEncryption.ServerSideEncryptionConfiguration`
  with an `SSEAlgorithm` of `AES256` (SSE-S3) for general workloads, or
  `aws:kms` with a customer-managed `KMSMasterKeyID` for regulated data.
- **Public access block** — every bucket should set
  `PublicAccessBlockConfiguration` with `BlockPublicAcls`, `IgnorePublicAcls`,
  `BlockPublicPolicy`, and `RestrictPublicBuckets` all `true`. These are the
  hard guardrails that prevent ACL/policy misconfiguration from publishing
  data to the internet.
- **Canned ACLs** — never use `PublicRead` or `PublicReadWrite` for
  `AccessControl`. Use bucket policies and `PublicAccessBlockConfiguration`
  to express access narrowly.
- **Defense in depth** — block public access at the account level
  (`AWS::S3::PublicAccessBlock`) so individual misconfigurations cannot
  override the org policy.
